# «ьё» — текущее рабочее устройство проекта

## Что это

«ьё» — мобильный сайт-колода на `HTML + CSS + JS`.

Это не каталог и не обычная галерея. Основной сценарий здесь один:

- пользователь выбирает одну из 5 колод;
- первый знак выходит сразу;
- дальше из колоды собирается расклад на 3 карты;
- после полного расклада пользователь читает три образа как `исток`, `узел`, `след`.

Запасной сценарий:

- поиск по одному слову или тегу;
- найденные работы показываются по одной карте;
- карту можно листать дальше.

## Что важно для текущей версии

- Основная аудитория: мобильные браузеры.
- Сайт должен оставаться рабочим и аккуратным и на десктопе.
- Интерфейс должен быть почти без кнопок.
- Пользователь нажимает на очевидные зоны:
  - на колоду;
  - на одну из трех верхних карт;
  - на большую активную карту;
  - на строку поиска;
  - на тег.

## Чего в проекте больше нет

В текущей концепции не используются:

- переворот карты;
- задняя сторона карты с метаданными;
- TTS-озвучка голосом;
- дополнительный экран действий после завершения расклада;
- постоянные панели режимов и сложная навигация.

Если что-то из этого снова появляется в коде или текстах, это артефакт старой концепции.

## Текущая структура опыта

### 1. Главная

На главной:

- логотип;
- небольшая стопка карт как центральный образ;
- короткий текст;
- список из 5 колод;
- строка поиска внизу.

Поведение:

- тап по колоде из списка колод запускает выбранный расклад;
- тап по строке поиска переводит в режим поиска.

### 2. Расклад

Расклад строится без дополнительных контролов:

- первая карта уже показана после входа в колоду;
- тап по большой карте открывает следующий знак, пока их не станет три;
- когда все три знака вышли, верхние маленькие карты выбирают активную позицию;
- большая карта после завершения расклада ведет по кругу между уже вышедшими картами.

Позиции расклада:

- `исток`
- `узел`
- `след`

### 3. Чтение

Текст живет вне карты.

В инфопанели показываются:

- позиция текущей карты в раскладе;
- название;
- принадлежность к колоде и тон;
- короткое описание;
- краткая интерпретация позиции;
- после третьей карты — общий смысл расклада.

### 4. Поиск

Поиск локальный, эвристический, без внешнего backend.

Основа:

- строка поиска;
- быстрые теги;
- scoring по тегам, названиям, описаниям и синонимическим группам.

Поведение:

- ввод слова сразу перестраивает выдачу;
- свайп или тап по большой карте переключает следующий результат;
- пустой поиск не должен ломать экран.

## Визуальная система

- почти черный фон;
- деликатный шум;
- неоновый акцент только в логотипе и активных состояниях;
- мягкое цветовое свечение от активной карты;
- карты ощущаются как физический объект, а не как карточки каталога.

Важно:

- не делать белых плашек;
- не раздувать интерфейс служебными кнопками;
- не превращать сайт в продуктовый каталог;
- держать темп спокойным и камерным.

## Данные и отбор изображений

Используются только заранее отобранные вертикальные работы из `assets/galleries`.

Кураторские правила:

- не использовать материалы из `/Users/ilyagmirin/Yandex.Disk.localized/Изображения/карты Таро`;
- не использовать реальные фото, если они ломают образ колоды;
- не использовать изображения с обильными встроенными надписями;
- не использовать слабые, пустые или выбивающиеся по тону работы;
- при спорных случаях допустим ручной кураторский пересмотр.

## Техническая схема

Ключевые файлы:

- `index.html`
- `styles.css`
- `app.js`
- `modules/catalog.js`
- `modules/copy.js`
- `modules/deck.js`
- `modules/gestures.js`
- `modules/renderer.js`
- `modules/search.js`
- `modules/theme.js`
- `modules/audio.js`

`audio.js` в текущей версии отвечает только за фоновую музыку, не за голос.

## Что считать хорошим состоянием проекта

Проект в хорошем состоянии, если:

- главная выглядит тихо и ясно на телефоне;
- колода запускается без лишних действий;
- три карты читаются как один расклад;
- поиск не спорит с основным сценарием;
- тексты и код не ссылаются на удаленные функции вроде переворота и TTS;
- визуальный язык остается единым.
